# Machine-Generated Text Detection

This repository contains experiments for detecting machine-generated text in a multilingual setting, plus a small FastAPI service for running the best checkpoint locally.

The task follows **SemEval 2024 Task 8, Subtask A**: given a text, predict whether it was written by a human or generated by a language model.

## Problem

**Input:** multilingual text sample  
**Output:** binary label:

- `0` — human-written text
- `1` — machine-generated text

The task is a supervised binary classification problem. The difficult part is not the label space, but the data: texts come from several languages and domains, and generated samples are often close to human writing in surface style.

## Dataset

The experiments use the multilingual split from SemEval 2024 Task 8, Subtask A.

Used split:

| Split | Samples |
|---|---:|
| Train, original | 12,000 |
| Validation | 3,000 |
| Train, after long-text splitting | 204,274 fragments |

The original dataset is not stored in this repository. Expected local paths are described in [`data/README.md`](data/README.md).

## Approach

I compared several multilingual encoders under the same classifier architecture.

### Models

| Encoder | Hugging Face model |
|---|---|
| XLM-RoBERTa base | `xlm-roberta-base` |
| mDeBERTa-v3 base | `microsoft/mdeberta-v3-base` |
| MiniLM multilingual | `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` |

### Classification head

For each model, token embeddings are mean-pooled over non-padding tokens and passed to a small MLP:

```text
Transformer encoder → mean pooling → dense layers → GELU → dropout → binary logits
```

### Training setup

I ran two training variants:

1. **Default**
   - standard cross-entropy loss
   - original texts

2. **Processed / article-style**
   - text cleaning
   - long-text splitting
   - label smoothing
   - staged fine-tuning

The staged schedule was:

```text
epoch 1: classifier head only
epochs 2-4: full encoder fine-tuning
epoch 5: classifier head only
```

Training logs include loss and validation metrics for TensorBoard.

## Results

The best run was **mDeBERTa-v3 with preprocessing and long-text splitting**.

| Experiment | Train size | Accuracy | F1 | Machine precision | Machine recall |
|---|---:|---:|---:|---:|---:|
| `mdeberta_processed_article` | 204,274 | 0.7627 | **0.7757** | 0.7354 | 0.8207 |
| `xlm_roberta_processed_article` | 204,274 | 0.6987 | 0.7355 | 0.6554 | 0.8380 |
| `xlm_roberta_original_default` | 12,000 | 0.6290 | 0.7038 | 0.5857 | 0.8813 |
| `minilm_original_default` | 12,000 | 0.6283 | 0.6853 | 0.5942 | 0.8093 |
| `mdeberta_original_default` | 12,000 | 0.6073 | 0.6749 | 0.5758 | 0.8153 |
| `minilm_processed_article` | 204,274 | 0.5320 | 0.3377 | 0.5774 | 0.2387 |

## Interpretation

Preprocessing and long-text splitting helped the larger encoders, especially mDeBERTa-v3 and XLM-RoBERTa. The best run also had the most useful precision/recall balance among the tested models.

MiniLM is lighter, but it did not handle the processed setup well. Its recall on the machine-generated class dropped sharply, which points to underfitting or instability with this training recipe.

## FastAPI inference service

The repository also includes a FastAPI app for serving a saved Hugging Face-compatible checkpoint.

### API endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/health` | Service health check |
| `POST` | `/predict` | Classify input text |

Input example:

```json
{
  "text": "This text should be classified as human-written or machine-generated."
}
```

Response example:

```json
{
  "label": "machine",
  "probability_human": 0.18,
  "probability_machine": 0.82
}
```

### Save the fine-tuned model

After training the best model in the notebook, save the model and tokenizer locally:

```python
model.save_pretrained("models/mdeberta_processed_article")
tokenizer.save_pretrained("models/mdeberta_processed_article")
```

By default, the API loads the model from:

```text
models/mdeberta_processed_article
```

To use another path, set `MODEL_PATH`:

```bash
export MODEL_PATH="models/mdeberta_processed_article"
```

### Run locally

Install dependencies:

```bash
pip install -r requirements.txt
```

Start the API:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Swagger UI:

```text
http://localhost:8000/docs
```

Example request:

```bash
curl -X POST "http://localhost:8000/predict" \
  -H "Content-Type: application/json" \
  -d @examples/request.json
```

### Run with Docker

Build the image:

```bash
docker build -t text-detection-api .
```

Run the container and mount local model files:

```bash
docker run -p 8000:8000 \
  -v $(pwd)/models:/app/models \
  text-detection-api
```
